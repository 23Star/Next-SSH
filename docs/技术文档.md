# Next-Panel 技术文档

## 一、项目概述

Next-Panel（轻量级无部署企业级 AI 运维平台）是一款基于 Electron 框架构建的新一代智能运维管理系统，将大语言模型（LLM）AI Agent 能力与 SSH 远程运维深度融合，实现了"连接即智能"的开箱即用体验。项目涵盖主进程（22 个模块）、渲染进程（28 个组件/页面）、预加载桥接层以及独立的 AI Agent 引擎四大架构层级，代码总量超过 17,000 行，支持 Windows/macOS/Linux 三平台部署。

## 二、系统架构设计

### 2.1 整体架构

系统采用四层隔离架构：

**主进程层（Main Process）**：基于 Electron 28.x 运行时，负责窗口生命周期管理、IPC 通信注册、SSH 连接池调度、SQLite 数据库持久化以及静态资源服务器托管。包含 22 个 TypeScript 模块，覆盖 SSH 连接管理（ssh2 协议栈）、环境配置持久化（better-sqlite3）、终端会话管理（node-pty）、文件系统操作、AI 配置存储、多语言国际化（中/英/俄）等核心能力。

**渲染进程层（Renderer Process）**：基于 React 18 + Vite 5 构建的现代化单页应用，包含 28 个 TSX/TS 模块。分为 Shell 层（Topbar 导航栏、HostPicker 主机选择器、AIDrawer 智能助手侧边栏）、Pages 层（Dashboard、Files、Firewall、Settings 等）、Components 层（Card、Gauge、ProgressBar、ChatInput、ToolUseCard 等）以及 Lib 层（useAgent、useConnection、useSystemSnapshot 等自定义 Hook）。

**预加载桥接层（Preload）**：通过 Electron 的 contextIsolation 机制暴露类型安全的 IPC API（window.electronAPI），包含 environment、terminal、chat、explorer、locale、theme、aiSettings 七大命名空间共 40+ 个方法。

**AI Agent 引擎**：基于 ReAct（Reasoning + Acting）模式的自治代理系统，包含 Agent Loop（多轮对话循环）、Tool Registry（工具注册表）、Permission Manager（权限审批）、Provider Adapter（LLM 适配器）四大子系统。内置 Bash、ListDir、ReadFile、SystemInfo 四个预置工具，支持动态扩展。

### 2.2 通信架构

渲染进程通过 Preload Bridge 发起 IPC 调用 → 主进程 Handler 处理业务逻辑 → SSH2 协议栈执行远程操作。AI 对话采用 SSE 流式传输，支持 content/thinking/done/error 四种事件类型，实现实时逐字输出与思维链可视化。

### 2.3 数据持久化

采用 better-sqlite3 嵌入式数据库，五张核心数据表：Environment（SSH 连接配置，AES 加密凭据）、ChatSession（AI 对话会话）、ChatContext（对话上下文消息，含思维链）、ServerOutput（服务器输出缓存）、Settings（键值对配置）。单例连接模式，写操作通过事务保证原子性。

## 四、核心模块技术实现

### 4.1 AI Agent 自治代理系统

Agent 系统采用 ReAct 模式，支持多轮工具调用循环。System Prompt 根据当前连接主机动态构建，注入 hostname、OS、CPU/内存/磁盘等实时系统快照，使 AI 基于准确上下文进行运维诊断。工具调用采用 JSON Schema 描述参数结构，执行前通过 Permission Dialog 确认。流式响应支持思维链展开/折叠展示，帮助用户理解 AI 推理过程。

**工具注册表**：
- **Bash**：远程命令执行（参数：command: string），支持超时控制
- **ListDir**：目录遍历（参数：path: string），返回文件/目录列表
- **ReadFile**：文件读取（参数：path: string），返回文件内容
- **SystemInfo**：系统信息采集（无参数），返回 CPU/内存/磁盘/负载等 12 项指标

### 4.2 SSH 连接管理引擎

基于 ssh2 协议栈，支持密码与公钥认证。连接建立后通过 node-pty 创建远程 Shell 通道，支持窗口自适应调整、数据双向流式传输。连接池模式复用已有连接，keepalive 心跳检测断线，凭据 AES-256 加密存储。

### 4.3 实时系统监控面板

通过 SSH 远程执行系统信息采集命令，解析为 SystemSnapshot 数据模型。自研环形仪表盘（Gauge，SVG + stroke-dashoffset）和进度条组件，颜色梯度映射（normal → warn → danger）。SWR 双层缓存策略：先展示缓存数据，后台异步更新。

### 4.4 文件管理器

SFTP 协议远程文件操作，左右分栏布局（文件树 + Monaco Editor 预览）。支持目录遍历、上传/下载、重命名、删除、语法高亮、差异对比。

### 4.5 防火墙管理

支持 iptables/nftables/ufw 三种后端，通过 SSH 查询命令解析规则并以表格展示。支持 UFW 状态查询、监听端口列表、nftables 规则链遍历。

## 五、安全设计

1. **进程隔离**：contextIsolation=true、nodeIntegration=false
2. **凭据加密**：AES-256 对称加密存储 SSH 密码/私钥
3. **API Key 安全**：UI 层掩码展示，原始密钥仅主进程解密使用
4. **权限审批**：AI Agent 远程命令执行需用户显式确认，前端弹窗 + 后端校验
5. **输入校验**：IPC 边界进行类型校验和合法性验证
6. **安全链接**：仅允许 https:// 外部链接通过 shell.openExternal 打开

## 六、跨平台与国际化

通过 electron-builder 实现三平台打包（Windows NSIS、macOS DMG、Linux DEB）。响应式布局（1100px/720px 断点）。CSS Design Tokens 支持一键切换。多语言支持中文/英文/俄文，运行时动态切换无需重启。
