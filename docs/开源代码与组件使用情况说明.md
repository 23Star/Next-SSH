# 开源代码与组件使用情况说明

## 一、项目开源信息

- **项目名称**：Next-Panel
- **开源协议**：MIT License
- **代码仓库**：GitHub 托管
- **代码总量**：17,000+ 行（不含 node_modules）
- **自主开发比例**：核心业务代码 100% 自主开发

本项目所有核心业务逻辑（主进程模块、渲染进程组件、AI Agent 引擎、Preload 桥接层）均为团队自主设计与开发。项目使用了若干开源组件和第三方库作为基础设施，以下为详细的使用情况说明。

## 二、核心依赖使用情况

### 2.1 桌面应用框架

| 组件 | 版本 | 许可协议 | 用途 | 使用方式 |
|------|------|---------|------|---------|
| Electron | 28.3.3 | MIT | 桌面应用运行时 | 框架级依赖，提供跨平台桌面应用能力 |
| electron-builder | 24.9.1 | MIT | 应用打包分发 | 开发工具，生成 Windows/macOS/Linux 安装包 |
| electron-rebuild | 3.2.9 | MIT | 原生模块编译 | 开发工具，自动编译 Node.js 原生模块 |

**使用说明**：Electron 是本项目的基础运行时框架，提供窗口管理、进程隔离、系统 API 访问等桌面应用核心能力。electron-builder 用于生产环境的跨平台打包，electron-rebuild 用于自动化编译含有 C++ 原生代码的 npm 模块。团队未修改上述组件的源代码。

### 2.2 前端框架与构建工具

| 组件 | 版本 | 许可协议 | 用途 | 使用方式 |
|------|------|---------|------|---------|
| React | 18.2.0 | MIT | UI 组件框架 | 渲染进程的组件化开发框架 |
| React DOM | 18.2.0 | MIT | React DOM 渲染 | React 的 DOM 渲染适配器 |
| Vite | 5.0.8 | MIT | 前端构建工具 | 开发服务器（HMR）与生产构建 |
| @vitejs/plugin-react | 4.2.1 | MIT | Vite React 插件 | JSX 转换与 React Refresh 支持 |

**使用说明**：React 作为渲染进程的 UI 框架，用于构建组件化的用户界面。Vite 作为构建工具提供极速的开发体验（HMR 热更新 <50ms）。TypeScript 为全栈代码提供静态类型检查。团队未修改上述组件的源代码。

### 2.3 SSH 与终端

| 组件 | 版本 | 许可协议 | 用途 | 使用方式 |
|------|------|---------|------|---------|
| ssh2 | 1.15.0 | MIT | SSH 协议实现 | 核心：实现 SSH 连接、Shell、SFTP、Exec |
| xterm.js | 5.3.0 | MIT | 终端模拟器 | 核心：浏览器端终端渲染引擎 |
| @xterm/addon-fit | 0.11.0 | MIT | xterm 自适应插件 | xterm.js 的 FitAddon，终端窗口自适应 |
| node-pty | 1.1.0 | MIT | 伪终端 | 核心：创建 PTY 进程，本地终端模拟 |

**使用说明**：ssh2 是 Node.js 生态中最成熟的纯 JavaScript SSH-2 协议实现，本项目使用其 Client、Shell、SFTP 和 Exec 能力建立远程连接。xterm.js 用于浏览器端的终端渲染，配合 node-pty 创建的伪终端实现完整的终端交互体验。团队未修改上述组件的源代码。

### 2.4 数据存储

| 组件 | 版本 | 许可协议 | 用途 | 使用方式 |
|------|------|---------|------|---------|
| better-sqlite3 | 9.2.2 | MIT | 嵌入式数据库 | 核心：本地数据持久化（5 张数据表） |

**使用说明**：better-sqlite3 是 SQLite3 的 Node.js 绑定，提供同步 API 的高性能嵌入式数据库。本项目使用其存储 SSH 连接配置、AI 对话历史、用户设置等数据，所有凭据字段采用 AES 加密。团队设计了完整的数据库 Schema 和数据访问层。

### 2.5 AI 能力

| 组件 | 版本 | 许可协议 | 用途 | 使用方式 |
|------|------|---------|------|---------|
| openai | 6.22.0 | Apache-2.0 | LLM SDK | 核心：AI Agent 的 LLM 通信层 |

**使用说明**：OpenAI SDK 用于与 LLM 服务端通信，支持流式输出（Streaming）和工具调用（Function Calling / Tool Use）。本项目通过 Provider Adapter 设计模式实现了多模型兼容——同一套 Agent 代码可适配 OpenAI GPT-4o、DeepSeek V3、通义千问 Qwen-Max 以及本地 Ollama 模型，用户在 Settings 页面自由切换。
