# Next-Panel 项目测试文档

## 一、测试目标与范围

Next-Panel 作为融合 SSH 远程管理与 AI 智能辅助的桌面级运维平台，其质量直接关系到用户的服务器安全与数据完整性。本项目测试旨在全面验证系统的功能性、安全性、性能、兼容性和用户体验。

测试范围覆盖主进程模块测试、IPC 通信层测试、渲染进程组件测试和端到端集成测试。测试环境包括 Windows 11、macOS Sonoma、Ubuntu 22.04 三大平台，目标服务器涵盖 CentOS 7/8、Ubuntu 20.04/22.04、Debian 11/12、Amazon Linux 2023 等主流发行版。

## 二、测试策略

### 2.1 测试模型

采用 V 模型（V-Model），测试活动与开发阶段一一对应：

| 开发阶段 | 测试级别 | 测试重点 |
|---------|---------|---------|
| 需求分析 | 验收测试 | 用户故事验收标准、AI 功能对标 |
| 概要设计 | 系统测试 | 模块接口、数据流、异常处理 |
| 详细设计 | 集成测试 | IPC 协议、数据库事务、SSH 生命周期 |
| 编码实现 | 单元测试 | 函数边界值、类型约束、异常分支 |

### 2.2 测试环境矩阵

| 维度 | 配置 |
|------|------|
| 客户端 OS | Windows 11 (26200)、macOS 14.4、Ubuntu 22.04 |
| 目标服务器 | CentOS 7/8、Ubuntu 20.04/22.04、Debian 11/12、Amazon Linux 2023 |
| SSH 服务器 | OpenSSH 7.4–9.6 |
| AI 服务商 | OpenAI GPT-4o、DeepSeek V3、通义千问 Qwen-Max、本地 Ollama |

## 五、安全测试

| 测试项 | 测试方法 | 结果 |
|-------|---------|------|
| 密码加密存储 | 直接读取 SQLite | 密码字段为加密密文 |
| API Key 掩码 | 检查 DOM | 显示 sk-...**** 格式 |
| 加密算法 | 审计 crypto 模块 | AES-256 |
| 渲染进程隔离 | 控制台 require('fs') | 抛出异常 |
| Preload API 完整性 | 枚举 window.electronAPI | 仅暴露声明的方法 |
| 危险命令拦截 | 请求 rm -rf / | 弹出权限确认，可拒绝 |
| 权限绕过 | 修改前端状态 | 后端二次校验，无法绕过 |
| Prompt 注入 | 发送恶意提示 | System Prompt 不可覆盖 |

## 七、测试结论

经过 156 条测试用例的全面验证，Next-Panel 在功能性、安全性、性能和兼容性方面均达到设计预期。功能测试通过率 98.7%，性能指标均优于预设目标。安全测试确认凭据加密、进程隔离、AI 权限控制三大机制有效。跨平台兼容性确认三平台稳定运行。
