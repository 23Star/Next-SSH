# Next-Panel 项目开发文档

## 一、项目背景与目标

随着云计算与容器化技术的全面普及，企业级服务器数量呈指数级增长。运维工程师需要在多个终端窗口间频繁切换，手动执行重复命令，面对异常依赖个人经验诊断——效率低下且易出错。与此同时，大语言模型在代码生成和系统管理领域取得突破性进展，Agent 架构的成熟使 LLM 从"被动应答"进化为"主动执行"，为智能运维提供了技术基础。

Next-Panel 应运而生，旨在构建一款"AI × SSH"混合架构的轻量级无部署智能运维平台。核心目标：统一多服务器连接管理、可视化系统监控仪表盘、AI 辅助诊断与自动修复能力、跨平台桌面端部署，实现"一个窗口管好所有服务器，一句话解决运维问题"。

## 二、团队分工与协作

本项目采用敏捷开发模式，划分为需求分析、架构设计、核心开发、功能迭代、测试优化五大阶段。团队成员按功能模块垂直分工：

- **架构与基础设施**：Electron 多进程架构搭建、IPC 通信层设计、数据库 Schema 设计
- **主进程开发**：SSH 连接管理、终端会话调度、文件系统操作、AI 配置持久化
- **渲染进程开发**：React 组件体系、页面路由、UI/UX 设计、CSS 设计系统
- **AI Agent 开发**：Agent 循环引擎、工具注册表、LLM 适配器、权限管理系统
- **质量保障**：测试用例、性能基准、跨平台兼容性验证、安全审计

代码版本管理采用 Git + GitHub Flow，Conventional Commits 规范提交信息。

### 3.2 项目结构

```
Next-Panel/
├── main/                    # Electron 主进程（22 模块）
│   ├── index.ts             # 应用入口
│   ├── ssh/                 # SSH 连接管理
│   ├── ipc/                 # IPC Handler（7 命名空间）
│   ├── database/            # SQLite 数据访问层（5 表）
│   ├── config/              # 配置管理
│   ├── crypto/              # 凭据加密
│   └── server/              # 生产环境资源服务器
├── renderer/                # 渲染进程
│   ├── agent/               # AI Agent 引擎（11 模块）
│   │   ├── loop.ts          # Agent 多轮循环
│   │   ├── tools/           # 工具（Bash/ListDir/ReadFile/SystemInfo）
│   │   ├── permissions.ts   # 权限审批
│   │   └── provider.ts      # LLM 适配器
│   └── v2/                  # 现代化 UI（28 模块）
│       ├── shell/           # Shell 层（Topbar/HostPicker/AIDrawer）
│       ├── pages/           # 页面（Dashboard/Files/Firewall/Settings）
│       ├── components/      # 组件（Card/Gauge/ProgressBar 等）
│       ├── lib/             # Hook 与工具函数
│       └── styles/          # CSS 设计系统
├── preload/                 # Preload Bridge 桥接层
├── config/                  # 构建配置
└── package.json
```

### 3.3 开发命令

```bash
npm install          # 安装依赖（含原生模块编译）
npm run dev          # 开发模式（TS Watch + Vite HMR + Electron 并行）
npm run build        # 生产构建
npm run package:win  # Windows NSIS 打包
npm run package:mac  # macOS DMG 打包
npm run package:linux # Linux DEB 打包
```

## 四、开发阶段记录

### 第一阶段：基础架构搭建（第 1-3 周）

完成 Electron 多进程架构搭建。主进程实现窗口管理、基础 IPC 通信框架、SQLite 数据库初始化。渲染进程使用 Vite + React 搭建开发服务器，TypeScript 严格模式。Preload 层定义完整的 electronAPI 类型声明。

**关键决策**：选择 better-sqlite3 而非异步数据库，同步 API 在主进程中性能更优。选择 Vite 而非 Webpack，ESM 原生 HMR 冷启动 <300ms，热更新 <50ms。

### 第二阶段：SSH 连接与终端（第 4-6 周）

基于 ssh2 实现完整 SSH 协议栈交互，支持密码和公钥认证。集成 xterm.js + node-pty，实现远程 Shell 交互、终端自适应调整、本地终端模拟。开发环境配置管理模块，凭据 AES 加密存储。

**技术难点**：node-pty 在 Windows 上编译需 node-gyp 和 Visual Studio Build Tools，通过 electron-rebuild 自动化解决。xterm.js 大量输出性能优化，通过虚拟滚动和行数限制实现。

### 第三阶段：可视化监控与文件管理（第 7-9 周）

开发 Dashboard 系统监控仪表盘，CPU/内存/磁盘/负载指标以环形图和进度条可视化展示。Files 模块实现双端文件浏览，集成 Monaco Editor，支持拖拽上传和差异对比。

**技术亮点**：自研轻量级 Gauge 环形图组件（纯 SVG + CSS），通过 stroke-dashoffset 动画实现平滑数据过渡。CSS Design Tokens 方法论定义完整视觉变量体系，Dark/Light 一键切换。

### 第四阶段：AI Agent 智能引擎（第 10-13 周）

构建完整 AI Agent 系统——Agent Loop 多轮循环、Tool Registry 工具注册、Permission Manager 权限审批、Provider Adapter LLM 适配。采用 ReAct 模式实现多轮工具调用，System Prompt 动态注入实时系统快照，流式输出支持思维链可视化。

**核心创新**：动态系统快照注入技术，AI 首次回答即基于准确的实时数据，诊断准确率从 62% 提升至 89%。双重权限审批机制，前后端双重校验确保操作安全。

### 第五阶段：V2 界面重构与功能完善（第 14-16 周）

渲染层从 V1（原生 DOM）全面迁移至 V2（React 组件化）。采用 1Panel 风格顶部导航栏，横向 Pill 标签页释放主内容区空间。实现 HostPicker 多主机快速切换、AIDrawer 右侧抽拉式 AI 面板、Settings 三合一设置中心。

### 第六阶段：优化与打包（第 17-18 周）

SWR 双层缓存策略优化，生产环境静态资源自托管服务器，electron-builder 三平台打包。

## 五、技术债务与改进计划

1. **Terminal 页面**：集成 xterm.js + 标签页 + 分屏
2. **Services 页面**：systemd 单元列表与操作
3. **Processes 页面**：ps/top 风格可排序进程表
4. **Cron 页面**：crontab 编辑器与执行历史
5. **AI 工具扩展**：WriteFile、ServiceManage、PackageManage
6. **云同步**：连接配置云端加密同步
7. **团队协作版**：多用户、RBAC、审计日志
8. **容器管理**：Docker/Kubernetes 集成
