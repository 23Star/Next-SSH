# 作品安装说明

## 一、运行环境要求

### 1.1 客户端环境

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10 22H2 / macOS 12 Monterey / Ubuntu 20.04 LTS | Windows 11 / macOS 14 Sonoma / Ubuntu 22.04 LTS |
| 处理器 | x64 架构，双核 | x64 架构，四核及以上 |
| 内存 | 4 GB RAM | 8 GB RAM 及以上 |
| 磁盘空间 | 500 MB（应用 + 依赖） | 1 GB |
| 网络 | 可访问目标 SSH 服务器 | 宽带网络（AI 功能需互联网连接） |

### 1.2 目标服务器环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（CentOS 7+、Ubuntu 18.04+、Debian 10+、Amazon Linux 2+） |
| SSH 服务 | OpenSSH 7.0+（启用 SSH-2 协议） |
| 认证方式 | 密码认证 或 公钥认证（RSA/Ed25519） |
| 网络连通 | 客户端可访问服务器 SSH 端口（默认 22） |

### 1.3 AI 功能（可选）

如需使用 AI 智能助手功能，需准备以下任一 LLM 服务的 API Key：

| 服务商 | 获取方式 | 推荐模型 |
|-------|---------|---------|
| OpenAI | https://platform.openai.com/api-keys | gpt-4o |
| DeepSeek | https://platform.deepseek.com/api_keys | deepseek-chat |
| 通义千问 | https://dashscope.console.aliyun.com/ | qwen-max |
| Ollama 本地 | https://ollama.ai 本地部署 | qwen2.5-coder:7b |

## 二、安装方式

### 方式一：下载预编译安装包（推荐）

适用于普通用户，无需安装 Node.js 等开发工具。

#### Windows

1. 下载 `Next-Panel-Setup-{version}.exe` 安装程序
2. 双击运行安装程序
3. 按照安装向导选择安装目录
4. 安装完成后从桌面快捷方式或开始菜单启动 Next-Panel

#### macOS

1. 下载 `Next-Panel-{version}.dmg` 镜像文件
2. 双击打开 DMG 文件
3. 将 Next-Panel 图标拖拽至 Applications 文件夹
4. 从 Launchpad 或 Applications 目录启动 Next-Panel

> 首次启动时，macOS 可能提示"无法验证开发者"。右键点击应用 → 选择"打开" → 在弹窗中点击"打开"即可。

#### Linux

```bash
# 下载 DEB 包
wget https://github.com/23Star/Next-SSH/releases/latest/download/next-panel_{version}_amd64.deb

# 安装
sudo dpkg -i next-panel_{version}_amd64.deb

# 如有依赖缺失
sudo apt-get install -f

# 启动
next-panel
```

### 方式二：从源码构建

适用于开发者或需要自定义修改的高级用户。

#### 步骤一：安装基础环境

```bash
# 安装 Node.js（推荐使用 nvm 管理版本）
nvm install 20
nvm use 20

# 验证版本
node --version   # v20.x.x
npm --version    # 10.x.x
```

#### 步骤二：克隆代码

```bash
git clone https://github.com/23Star/Next-SSH.git
cd Next-SSH
```

#### 步骤三：安装依赖

```bash
npm install
```

> 注意：安装过程中会自动触发 `electron-rebuild` 编译 node-pty、better-sqlite3 等原生模块。Windows 平台需要安装 Visual Studio Build Tools（C++ 桌面开发工作负载）。macOS 平台需要安装 Xcode Command Line Tools（`xcode-select --install`）。

#### 步骤四：开发模式运行

```bash
npm run dev
```

此命令会并行启动以下三个进程：
- TypeScript Watch（主进程代码热编译）
- Vite Dev Server（渲染进程 HMR 热更新）
- Electron（应用运行）

#### 步骤五：生产构建与打包

```bash
# 构建生产版本
npm run build

# 打包安装程序
npm run package:win     # Windows NSIS 安装包
npm run package:mac     # macOS DMG 镜像
npm run package:linux   # Linux DEB 包
```

构建产物位于 `release/` 目录。

## 三、首次使用配置

### 3.1 添加 SSH 连接

1. 启动 Next-Panel
2. 点击顶部导航栏的 **Settings**（设置）标签页
3. 在 **SSH Hosts** 标签下点击 **Add Host** 按钮
4. 填写服务器信息：
   - **Name**：显示名称（可选，如"生产服务器-Web"）
   - **Host**：服务器 IP 地址或域名
   - **Port**：SSH 端口（默认 22）
   - **Username**：登录用户名
   - **Auth Type**：选择 Password（密码）或 Private Key（公钥）
   - 如选密码，填写 Password 字段
   - 如选公钥，填写 Private Key Path（如 `~/.ssh/id_rsa`）
5. 点击 **Test Connection** 验证连通性
6. 点击 **Add Host** 保存

### 3.2 连接服务器

1. 点击顶部导航栏右侧的 **主机选择器**（显示"No host selected"）
2. 从下拉菜单中选择已添加的主机
3. 连接建立后，状态指示器变为绿色，Dashboard 自动加载系统监控数据

### 3.3 配置 AI 助手（可选）

1. 进入 **Settings** → **AI Provider** 标签页
2. 填写配置信息：
   - **API URL**：LLM 服务端点（如 `https://api.deepseek.com/v1`）
   - **API Key**：您的 API 密钥
   - **Model**：模型名称（如 `deepseek-chat`）
3. 点击 **Test Connection** 验证 API 可用性
4. 点击 **Save** 保存配置
5. 点击右上角 **AI 助手按钮**（星形图标）打开 AI 对话面板，即可开始使用

### 3.4 快捷方式（顶部主机选择器）

在顶部导航栏的主机选择器下拉菜单底部，点击 **"Manage Hosts…"** 可快速跳转到 Settings 页面管理 SSH 连接。

## 四、常见问题

### Q1：Windows 安装依赖时报错 `node-gyp` 相关错误

**原因**：缺少 C++ 编译工具链。

**解决方案**：
```bash
# 安装 Visual Studio Build Tools
npm install -g windows-build-tools
# 或手动安装 Visual Studio Build Tools，勾选"C++ 桌面开发"工作负载
```

### Q2：macOS 提示"无法打开，因为无法验证开发者"

**解决方案**：
```bash
# 移除隔离属性
xattr -cr /Applications/Next-Panel.app
# 或在系统设置 → 隐私与安全性 → 点击"仍要打开"
```

### Q3：Linux 下 `npm install` 报错

**解决方案**：
```bash
# 安装编译依赖
sudo apt-get install build-essential libssl-dev python3-dev
# 然后重新安装
npm install
```

### Q4：AI 功能无响应

**检查项**：
1. 确认 API URL 和 API Key 填写正确
2. 确认网络可访问 LLM 服务端点
3. 进入 Settings → AI Provider 点击 Test Connection 检查连通性
4. 如使用 Ollama 本地模型，确认 Ollama 服务已启动（`ollama serve`）

### Q5：SSH 连接失败

**检查项**：
1. 确认服务器 IP 和端口正确
2. 确认客户端可 ping 通服务器
3. 确认 SSH 服务已启动（`systemctl status sshd`）
4. 确认认证方式与凭据正确
5. 在 Settings 页面点击 Test Connection 验证连通性

## 五、卸载说明

### Windows
控制面板 → 程序与功能 → 找到 Next-Panel → 卸载

### macOS
将 `/Applications/Next-Panel.app` 拖入废纸篓

### Linux
```bash
sudo dpkg -r next-panel
```

### 清理用户数据

应用数据存储在以下目录，卸载后可手动删除：

| 平台 | 数据目录 |
|------|---------|
| Windows | `%APPDATA%\next-ssh` |
| macOS | `~/Library/Application Support/next-ssh` |
| Linux | `~/.config/next-ssh` |
